### 用http或者https执行在线脚本的时候失败的问题

比如说你要执行这个脚本如下

```bash
sh -c "$(curl -fsSL https://raw.githubusercontent.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"
```

但是呢你开启了网络代理

链接上了良好的wifi 但是他卡住不动了
为啥会这样子

你的系统虽然网络代理可以让浏览器正常访问 但是你的bash默认不走这个代理

你可以在bash中修改变量 
```bash
export http_proxy=http://127.0.0.1:7897
export https_proxy=http://127.0.0.1:7897
export ALL_PROXY=http://127.0.0.1:7897
```
但是这样做退出这些变量就消失了 

可以检查当前的bash中代理环境变量是否生效

```bash
# 1. 查看变量是否存在
echo $http_proxy
echo $https_proxy
echo $ALL_PROXY

# 2. 查看所有代理相关环境变量
env | grep -i proxy

# 3. 实际测试：对比走代理和不走代理的速度
curl -I -w "Time: %{time_total}s\n" https://github.com
# 如果上面显示很快（<<1秒），说明代理生效了
```

直接写入到zshrc中

```bash
# 追加到 zshrc 文件末尾
echo 'export http_proxy=http://127.0.0.1:7897' >> ~/.zshrc
echo 'export https_proxy=http://127.0.0.1:7897' >> ~/.zshrc
echo 'export ALL_PROXY=http://127.0.0.1:7897' >> ~/.zshrc

```


```bash
# 刷新
source ~/.zshrc 


# 确认当前终端已生效
echo $http_proxy
```
